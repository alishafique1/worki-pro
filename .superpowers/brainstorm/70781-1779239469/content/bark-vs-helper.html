Bark.com vs The Helper — Core Model Difference

Understanding why Bark converts better, and what to steal

🐕 Bark.com flow
1. Homepage
Big search box ("What do you need?") + visual category cards with photos
2. Pick category
Service-first, not role-first. User thinks "I need a plumber" not "I am a consumer"
3. Dynamic questions
Each service has tailored follow-up questions ("What type of plumbing work?", "Urgent?", "Property type?")
4. Location
Postal code / city entered after questions
5. Contact info / Sign up
Account created LAST — after they're already invested in the flow
✓ Result: free quotes delivered
🏠 The Helper — current flow
1. Homepage
Nav links, hero copy, no clear "start here" moment
2. Login / OTP
User must create an account BEFORE explaining what they need
3. Role selection
Homeowner vs Service Pro (generic)
4. Profile info
Name, phone, postal code, SMS consent
5. Pick interests
Generic category grid — no tailored questions, optional, no context
✗ Now what? Dashboard is empty
The Three Problems You Identified
🏠
Homepage confusion

No clear "start here". Services exist but aren't discoverable. Dropdown nav doesn't show categories well.

🧩
Onboarding is static

Picking HVAC doesn't ask "is it AC repair or furnace install?" No dynamic follow-up. No service request created.

📄
Service detail pages are thin

Landing pages exist (/hvac, /plumbing) but don't have pricing, FAQs, local trust signals, or a strong CTA.

Bark's Key Insight — steal this
Service-first, account-second. Users don't sign up to "be on a marketplace." They sign up because they need something done right now. Bark captures intent immediately, then creates the account as a byproduct of the request. The Helper currently asks people to invest in an account before they've seen any value.
