## Edge Cases
1. **OTP rate limit**: Max 3 sends per 5 min ‚Üí error message. Max 5 attempts ‚Üí locked, re-send required.
2. **Double lead claim**: Same provider claims same lead twice ‚Üí `{ alreadyClaimed: true }`, no double charge
3. **Unverified provider**: Provider with `verificationStatus ‚âÝ VERIFIED` attempts claim ‚Üí error
4. **Unassigned lead review**: Submit review for NEW (unclaimed) lead ‚Üí blocked
5. **Duplicate review**: One review per serviceRequestId ‚Üí blocked with message
6. **Lead PII masking**: GET /provider/leads response must NOT contain name/phone/email before claim
7. **Auth redirects**: Unauthenticated user hits /dashboard ‚Üí redirected to /login
8. **Provider applies as CONSUMER**: Bad role ‚Üí blocked at signup level
9. **Empty states**: /my-requests with 0 requests ‚Üí empty state. /provider/leads with 0 leads ‚Üí empty state
10. **Password reset**: Request ‚Üí receive email ‚Üí reset link ‚Üí login with new password
