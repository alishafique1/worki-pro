Invalid transitions (all prevented by code):
- NEW → COMPLETED (skips assignment) → blocked by `assignedProviderId` null check

- COMPLETED → anything → terminal state, no transitions defined

5. **Job completed → review**: Consumer submits review → points issued → admin moderates

### Edge Cases to Verify

1. **OTP rate limiting**: 3 sends in 5 min → error. 5 incorrect attempts → locked.
2. **Lead claim idempotency**: Double-claim returns `{ alreadyClaimed: true }`, no double charge
3. **Unverified provider can't claim**: Provider with `verificationStatus ≠ VERIFIED` gets rejection
4. **Consumer can't review service they didn't book**: Authorization check on review creation
5. **Reward points on first completed job**: Signup bonus + completion bonus issued
6. **Admin moderation flow**: PENDING → PUBLISHED / REJECTED → provider rating recalculated
