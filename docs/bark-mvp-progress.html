Backend Operations (Wasp + Node.js)

9 new operations
sendOtp · verifyOtp — Phone Verification
Sends a 6-digit code via Twilio SMS. SHA-256 hashed storage. Auto-falls back to console.log in dev when Twilio keys are absent. Timing-safe comparison prevents brute-force.
Twilio SMS Security
getPublicLeadFeed — Masked Lead Feed
Returns NEW/QUALIFYING/QUALIFIED service requests scoped to the pro's registered service categories. Strips all PII (name, phone, email) from the payload. Provider cannot see contact until they claim.
PII Masking Bark Pattern
claimLead — Lead Unlock + Fee
Sets assignedProviderId, transitions status to ASSIGNED, creates a ProviderFee(QUALIFIED_LEAD, $5), seeds the message thread, and emails the consumer. Fully idempotent.
Revenue Event Idempotent
getPublicProvider — Public Pro Profile by Slug
Fetches a verified, active provider by URL slug. Includes categories and up to 30 published reviews. Used by the public-facing /pro-public/:slug page.
Public API SEO-ready
submitReview · getReviewsForProvider
One review per consumer per service request (duplicate guard). Automatically recomputes the provider's ratingInternal average via Prisma aggregate after each write.
Reviews Trust Signal
getMessagesForRequest — Unified Message Query
Returns CommunicationLog entries (INTERNAL_NOTE channel) for a request. Accessible to both the consumer and the assigned provider via ownership check.
Shared Query
getAdminReviews · moderateReview — Admin Moderation
Admin-gated operations to list all reviews and change status (PUBLISHED / REJECTED / PENDING). Recomputes provider rating on every moderation action.
Admin Moderation
